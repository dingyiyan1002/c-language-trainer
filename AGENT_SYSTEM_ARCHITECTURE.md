# 动画可视化智能体系统架构 v1.0

## 项目概述

**目标**: 为 700+ 道 C 语言题目创建交互式代码执行可视化动画

**技术基础**: Skills 交互式代码可视化教学动画制作系统 v4.0

**题目来源**:
- Level 1-5 嵌入式开发题库 (约 700 道)
- 覆盖知识点：变量、指针、数组、函数、内存、数据结构

---

## 一、智能体组织架构

```
┌─────────────────────────────────────────────────────────────┐
│                    总控智能体 (Main Controller)              │
│  - 任务分解与分配                                           │
│  - 进度跟踪与质量管理                                       │
│  - 最终验收测试                                             │
└─────────────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        │                     │                     │
        ▼                     ▼                     ▼
┌───────────────┐    ┌───────────────┐    ┌───────────────┐
│  开发智能体组  │    │  测试智能体组  │    │  集成智能体组  │
│  (1-200)      │    │  (1-50)       │    │  (1-10)       │
└───────────────┘    └───────────────┘    └───────────────┘
```

### 1.1 总控智能体 (Main Controller)

**职责**:
- 读取所有题目数据
- 按知识点分类题目
- 分配任务给开发智能体
- 收集并合并生成的可视化文件
- 执行最终质量检查
- 生成验收报告

**工作流程**:
```
1. 读取 src/data/lessons.ts 和 src/data/embeddedQuestions.ts
2. 统计所有题目 ID 和知识点类型
3. 按章节分配给开发智能体批次
4. 等待所有开发智能体完成
5. 调用测试智能体验证
6. 集成到项目并测试
7. 生成总结报告
```

### 1.2 开发智能体 (Developer Agents) - 200 个

**分组架构**:
```
开发智能体组 A (1-50)  : Level 1 基础题目 (80 道)
开发智能体组 B (51-100): Level 2 进阶题目
开发智能体组 C (101-150): Level 3 高级题目
开发智能体组 D (151-200): Level 4-5 专家题目
```

**单个开发智能体职责**:
- 接收 3-5 道题目
- 分析题目知识点类型
- 根据 skill 文档生成对应的 HTML 可视化文件
- 自测基本功能
- 返回生成的文件路径

**开发智能体输出模板**:
```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>题目{ID} - {知识点名称} - 可视化演示</title>
  <!-- 4 个 CDN -->
  <style>
    /* CSS 变量 */
    /* 全局样式 */
    /* @keyframes 动画库 (完整版) */
    /* 组件样式 */
  </style>
</head>
<body>
  <div id="root"></div>
  <script type="text/babel">
    // React 组件实现
    // 状态机
    // 导航控制
    // 可视化逻辑
  </script>
</body>
</html>
```

### 1.3 测试智能体 (QA Agents) - 50 个

**职责**:
- 验证 HTML 文件语法
- 检查动画库完整性
- 验证状态机逻辑
- 测试导航功能
- 检查响应式设计

**测试清单**:

- [ ] 响应式布局正常

### 1.4 集成智能体 (Integration Agents) - 10 个

**职责**:
- 将可视化文件集成到项目

### 2.2 可视化模板库

基于 skill 文档，预定义以下模板：

#### 模板 A: 变量内存可视化
```javascript
// 适用：变量定义、赋值、类型转换
可视化：内存卡片 (变量名、类型、值、地址)
动画：boxPop(创建)、valuePop(变化)
```

#### 模板 B: 指针关系可视化
```javascript
// 适用：指针定义、指针运算、指针与数组
可视化：卡片 + SVG 箭头
动画：pointerMove(箭头移动)、pulse(焦点)
```

#### 模板 C: 栈帧可视化
```javascript
// 适用：函数调用、递归、局部变量
可视化：栈帧盒子 (参数、局部变量、返回值)
动画：stackPush(入栈)、stackPop(出栈)
```

#### 模板 D: 数组/内存格可视化
```javascript
// 适用：数组、内存布局、memcpy/memset
可视化：连续内存格子
动画：memoryWrite(写入)、dataFlow(流动)
```

#### 模板 E: 位操作可视化
```javascript
// 适用：位运算、寄存器操作
可视化：8/16/32 位格子，每位可点击
动画：bitFlip(翻转)、signalPulse(脉冲)
```

## 八、错误处理与恢复

### 8.1 常见错误

| 错误类型 | 原因 | 解决方案 |
|---------|------|---------|
| HTML 解析失败 | 语法错误 | 调用修复智能体 |
| 动画不播放 | keyframe 缺失 | 检查动画库完整性 |
| 状态不同步 | 状态机逻辑错误 | 重新实现 useEffect |
| 导航失效 | 边界条件错误 | 检查 maxSteps 定义 |

### 8.2 恢复策略

```
1. 检测到错误 → 记录到错误日志
2. 分类错误类型 → 分配给对应修复智能体
3. 修复后重新测试 → 通过则合并，失败则升级
4. 严重错误 → 通知总控智能体人工介入
```
