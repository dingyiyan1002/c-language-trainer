- ✅ 常见问题

## 🔧 纠正类

### ⚠️ 跳过需求理解
```
⚠️ 请遵循 SOLO Coder 流程！

请先：
1. 确认需求理解
2. 输出实现规划

然后再开始实现。
```

### ⚠️ 代码不完整
```
⚠️ 请提供完整的、可运行的代码。

不要只给片段，请提供：
- 完整的代码
- 依赖说明
- 配置说明
- 运行方法
```

### ⚠️ 缺少测试
```
⚠️ 请提供测试验证方法。

包括：
- 测试用例
- 测试命令
- 预期结果
- 验证步骤
```

### ⚠️ 没有解释
```
⚠️ 请解释代码的关键部分。

特别是：
- 复杂逻辑
- 算法原理
- 设计决策
- 权衡考虑
```

记住这三句话，随时提升对话质量：

2. **"请提供完整的代码"** - 避免代码片段
3. **"请提供测试方法"** - 确保质量可靠
