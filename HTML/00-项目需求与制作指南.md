### 项目目标
将 718 道 C 语言题目转化为**交互式可视化动画网页**，每道题一个独立的 HTML 文件，可直接在浏览器中运行教学。

└──────────────┬──────────────────────┘
               │
               ↓ 数据文件
┌─────────────────────────────────────┐
│ 人工验证的引擎 (锁定)                │

│ (引擎 + 数据)                       │

### 关键原则

✅ **引擎锁定原则**: engine_v1.html 由人工提供并验证，AI **绝对不要修改或重新生成**

✅ **数据分离原则**: AI **只生成 .js 数据文件**，不包含任何渲染逻辑

✅ **组装原则**: 最终 HTML = engine_v1.html + 数据文件内容

## ⚠️ 历史问题与教训

### ❌ 错误的做法（AI 常犯的错误）

1. **AI 自己生成引擎** → 生成的 engine_v1.html 质量极差
   - 使用 Emoji（💭💡⚠️）- 明确禁止
   - CSS 类名错误（.code-editor vs .code-line-flow）
   - 布局错误（CSS Grid vs flex）
   - 缺少关键特性（毛玻璃效果、动画效果）
   - 执行光球定位错误
   - 控制栏位置错误

2. **AI 修改引擎** → 破坏了已验证的功能

3. **AI 生成整 HTML** → 包含渲染逻辑，违反分离原则

### ✅ 正确的做法

1. **AI 只生成数据** → .js 文件，仅包含 `window.LESSON_DATA` 对象
2. **引擎由人工锁定** → engine_v1.html 不可修改
3. **组装由脚本完成** → 自动化脚本将数据注入引擎

## 🎯 AI 的任务（Claude Code）

### 任务描述

**你是 C 语言可视化教学系统的自动化数据生产线。**

你的**唯一任务**是：读取题目数据，生成符合规范的 .js 数据文件。

### 输入

- **题目数据**: `src/data/chapters/*.ts` 文件中的 `lessons` 数组
- **规范文档**: `HTML/skills/工程 skills.md` 中的「模式 B — 数据生成」

### 输出

- **数据文件**: 每个题目生成一个 `.js` 文件到对应的 HTML 子目录
- **文件格式**: `window.LESSON_DATA = { ... };`

### 文件命名规范

```
{章节英文}_{三位序号}_{lesson.id}.js

示例:
- basics_test_001_bt-data-types.js
- level1_001_l1-hello-world.js
- level2_001_l2-pointer-basic.js
- interview_001_intv-stack-heap.js
```

## 🔧 数据生成铁律

**违反任何一条都会导致生成的数据无法使用！**

1. ✅ **代码来源**: 代码必须来自 `lesson.content`，禁止自己编造
2. ✅ **Token 化**: codeLines 每行必须正确 Token 化，禁止整行放一个 token
3. ✅ **初始场景**: scenes[0].line 必须是 -1
4. ✅ **变量快照**: 每个 scene.vars 必须是完整快照（包含所有已声明变量）
5. ✅ **执行行**: executedLines 不含当前高亮行，且随 step 单调递增
6. ✅ **行号对应**: scene.line 必须等于 explanation.text 描述的那行代码的行号
7. ✅ **颜色限制**: explanation.color 只能用 green/red/yellow/purple/blue/orange
8. ✅ **图标限制**: explanation.icon 只能用 ICON_MAP 中已有的键名
9. ✅ **详细解释**: explanation.detail 不能为空，必须写底层机制解释
10. ✅ **出现步数**: varConfig.appearStep 必须精确等于该变量首次出现在 vars 中的 step
11. ✅ **陷阱具体**: summary.pitfalls 必须是 3 条具体的易错点，禁止废话
12. ✅ **迁移问题**: summary.transferQuestion 必须与本题知识点相关

## 🚀 组装流程

### 方法 1：手动组装（测试用）

1. 打开生成的 `.js` 文件（如 `HTML/基础入门/basics_test_001_bt-data-types.js`）
2. 复制整个 `window.LESSON_DATA = { ... };` 内容
3. 打开 `HTML/engine_v1.html`
4. 找到 `window.LESSON_DATA = {};` 这一行
5. 用复制的内容替换它
6. 保存为新的 HTML 文件（如 `test-001.html`）
7. 浏览器打开测试

### 方法 2：自动化组装（批量生产）

```powershell
# PowerShell 脚本示例
$engine = Get-Content 'HTML/engine_v1.html' -Raw -Encoding UTF8
$jsFiles = Get-ChildItem 'HTML/基础入门/*.js'

foreach ($jsFile in $jsFiles) {
    $data = Get-Content $jsFile.FullName -Raw -Encoding UTF8
    $output = $engine -replace 'window\.LESSON_DATA\s*=\s*\{\s*\};', $data
    $outputName = $jsFile.BaseName + '.html'
    $output | Out-File "HTML/输出/$outputName" -Encoding UTF8
}
```
