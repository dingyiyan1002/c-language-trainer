# 窗口 1：基础入门 + Level1（257 题）

```
你是 C 语言可视化教学系统的自动化数据生产线。

【重要】引擎已由人工验证并锁定，你不要修改 engine_v1.html。
你只需要生成 .js 数据文件，每个文件的内容是：
window.LESSON_DATA = { meta:{...}, codeLines:[...], varConfig:[...], scenes:[...], summary:{...} };

引擎会自动读取这个对象并渲染。你不需要关心渲染逻辑。

【项目路径】
C:\Users\6\Downloads\web-development-request-received - 副本

【引擎文件】
HTML\engine_v1.html（已锁定，不要修改）

【Skills 规范】
HTML\skills\工程 skills.md（完整阅读，严格遵守其中「模式 B — 数据生成」的全部规则）

【你的任务】
自动读取以下章节文件中的每一个 lesson，为每题生成一个 .js 数据文件。

章节 1：src\data\chapters\basics_test.ts → 输出到 HTML\基础入门\
章节 2：src\data\chapters\level1.ts → 输出到 HTML\Level1\

【执行流程】

Step 1：读取 skills 文件，理解 LESSON_DATA 的完整结构规范

Step 2：读取章节 .ts 文件，解析 lessons 数组，获取每个 lesson 的 id、title、content、keyPoints

Step 3：对每个 lesson 执行以下操作：
  a. 从 lesson.content 中提取代码块（```c ... ```之间的内容）
  b. 如果代码不完整（缺 main/缺 include），补全为可编译程序
  c. 将代码逐行 Token 化（token.type 只能是 kw/pp/str/num/fn/op/cmt/id/punc/ws）
  d. 设计执行步骤（每个有意义的语句一个 step）
  e. 为每步编写 explanation（text + detail + actor + icon + color）
  f. 标注变量（varConfig，6 种颜色循环分配）
  g. 至少设置 1 个 prediction（预测暂停点）
  h. 至少设置 1 个 misconception（误区提示）
  i. 编写 summary（oneLiner + 3 条 pitfalls + transferQuestion + miniExercise）
  j. 组装为完整的 window.LESSON_DATA = { ... }; 

Step 4：保存文件
  文件名格式：{章节英文}_{三位序号}_{lesson.id}.js
  basics_test 章节示例：basics_test_001_bt-data-types.js
  level1 章节示例：level1_001_l1-hello-world.js

Step 5：每完成 1 个章节，在终端输出汇总：
  章节名 | 总题数 | 已完成 | 失败数

【数据生成铁律（违反任何一条立即停止修正）】

1. 代码必须来自 lesson.content，禁止自己编造
2. codeLines 每行必须正确 Token 化，禁止整行放一个 token
3. scenes[0].line 必须是 -1
4. 每个 scene.vars 必须是完整快照（包含所有已声明变量的当前值）
5. executedLines 不含当前高亮行，且随 step 单调递增
6. scene.line 必须等于 explanation.text 描述的那行代码的行号
7. explanation.color 只能用 green/red/yellow/purple/blue/orange
8. explanation.icon 只能用 ICON_MAP 中已有的键名：info/check/alert/bolt/rocket/lightbulb/memory/pointer/skull/stack/lock/folder/target/terminal/gear
9. explanation.detail 不能为空，必须写底层机制解释
10. varConfig.appearStep 必须精确等于该变量首次出现在 vars 中的 step
11. summary.pitfalls 必须是 3 条具体的易错点，禁止写"注意语法"之类废话
12. summary.transferQuestion 必须与本题知识点相关

【自检】
每题生成后自动检查：
  - scenes[0].line === -1
  - 每个 scene.vars 包含所有已声明变量
  - executedLines 不含 scene.line
  - explanation.color 在 6 个合法值内
  - 有至少 1 个 prediction
  - 有至少 1 个 misconception
  - summary 四个字段全部非空
不通过则立即修正后重新保存。

【现在开始】
先读取 skills 文件，然后从 basics_test.ts 的 lessons[0] 开始，逐题生成，全部保存完毕后再处理 level1.ts。
```

# 总览

```
窗口 0（已发）：生成 engine_v1.html（引擎）
窗口 1：basics_test.ts + level1.ts          → 257 题
窗口 2：level2-5 + interview + basics       → 284 题  
窗口 3：剩余 11 个小章节                       → 177 题

3 个窗口可以同时开，并行跑，互不影响。
每个窗口独立读取 skills + 对应章节文件 → 输出 .js 到对应目录。
全部跑完后，每个目录下的 .js 就是最终的数据文件。

组装方式：
  每个输出目录下的 .js 文件头部插入 engine_v1.html 的引擎代码
  或者直接把 .js 内容替换 engine_v1.html 中的 window.LESSON_DATA = {};
  即可得到最终可运行的 HTML 文件。
```
